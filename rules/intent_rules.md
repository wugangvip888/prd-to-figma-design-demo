# Intent Rules

## 固定字段

Intent 文件必须包含以下字段，且顺序固定：

```text
页面目标：xxx
用户进来第一件事：xxx
视觉重心：xxx
主要操作：xxx
次要目标：xxx
关键状态：xxx
PRD硬约束：xxx
用户心智：xxx
```

## 写作规则

```text
生成 Intent 前必须全盘阅读 PRD 内容，不得只读取当前小节或局部页面描述。
若 PRD 中存在【PRD-原型图】、页面原型图、线框图或 Markdown 字符原型，仍必须全盘阅读 PRD，不得只读取原型图或只读取局部说明；原型图仅在页面结构和可见表达与正文概述冲突时，作为 Intent 判断的最高优先级输入。
【PRD-原型图】在页面目标、用户首要任务、视觉重心、主要操作、次要目标、关键状态、可见文案和页面结构识别上优先于正文概述。
Intent 只描述页面意图和设计判断，不展开具体布局细节。
页面目标必须服务 PRD 中的核心业务任务。
用户进来第一件事必须描述用户进入页面后的首要动作。
视觉重心必须能指导后续元素权重排序。
主要操作用于识别 P0/P1。
次要目标用于识别 P2/P3。
关键状态只记录影响线框图表达的页面级状态。
PRD硬约束必须来自 PRD 原文，不自行扩展业务规则。
当【PRD-原型图】与 PRD 正文冲突时，页面结构、模块顺序、可见文案、状态表达和 CTA 位置优先遵循【PRD-原型图】；业务逻辑、字段约束、校验规则、数据模型、埋点和后台流程优先遵循 PRD 正文，并在 Intent 中标记冲突。
用户心智用于约束页面气质和信息组织方向。
语言应简短、明确，偏判断，不写长段说明。
```

## 强约束触发词

```text
必须、仅、默认、右侧、配套、图标、箭头、固定、隐藏、显示、不展示、无冗余、同步、独立。
```
